## Что уже работает

Локальный pipeline сформировал manifest, Docling JSON/Markdown, структурированные
таблицы, chunks, entities, SQLite FTS5 и RAG JSONL. Корпус не нужно повторно
отправлять в LLM или заново пропускать через Docling для запуска поиска.

В entities присутствуют:

- 13666 кодов МКБ-10
- 10607 уровней доказательности
- 310 лабораторных порогов
- 272 дозировки

## Обнаруженные ограничения

1. Neural embeddings отсутствуют. Поиск использует полный TF-IDF индекс, FTS5,
   medical entities и metadata filters.
2. Автоматические document families нельзя считать надежными. 560 документов
   входят в группы с общими OCR-заголовками; одна ошибочная группа содержит 338
   разных документов с названием «Клинический протокол диагностики и лечения».
3. Поэтому API использует `version_policy=all` по умолчанию. Режим
   `latest_known` является opt-in и исключает известные общие заголовки.
4. В pipeline-отчете есть четыре ошибки длинных имен файлов и три отсутствующих
   Markdown-артефакта.
5. База содержит описания шкал и формулы, но не является исполняемым движком
   калькуляторов. OCR-таблицы нельзя автоматически использовать для клинических
   расчетов без отдельной валидации.

## Граница проектов

`MEDICAL KNOWLEDGE BASE` отвечает только за поиск, фильтрацию, citations и RAG
evidence arrays. Проект `CALCULATOR BAZA` не изменяется и остается отдельным
вычислительным продуктом.
