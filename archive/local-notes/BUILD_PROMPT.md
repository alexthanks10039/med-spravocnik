# Исполняемый промт сборки iPhone-приложения

Создай производственный MVP Flutter-приложения для iPhone на основе двух существующих FastAPI-сервисов в рабочей папке:

- `CALCULATOR BAZA`: каталог медицинских калькуляторов, schema endpoint и выполнение расчётов;
- `MEDICAL KNOWLEDGE BASE`: полнотекстовый поиск по локальным клиническим протоколам с цитируемыми фрагментами.

Требования:

1. Создай отдельный Flutter-проект `medical_reference_app` только с iOS-платформой.
2. Используй Material 3 и русскоязычный интерфейс с тремя разделами: «Расчёты», «Протоколы», «Настройки».
3. Загружай каталог через `GET /api/v1/calculators`, поддерживай поиск через `GET /api/v1/search`.
4. Строй формы динамически через `GET /api/v1/calculators/{tool_id}/schema`, валидируй обязательность и диапазоны, отправляй данные в `POST /api/v1/calculate/{tool_id}`.
5. Показывай значение, единицы, интерпретацию, рекомендации и компоненты результата.
6. Ищи протоколы через `GET /api/search`; показывай документ, раздел, фрагмент, версию, дату и предупреждение о версии.
7. Позволь менять URL обоих API и проверять `/health`. Поддержи `--dart-define=CALCULATOR_API_URL=...` и `KNOWLEDGE_API_URL=...`.
8. Добавь клинический дисклеймер, не выдавай приложение за медицинское изделие.
9. Версию `0.0.0.01` представь в допустимом для Flutter/Apple формате `0.0.0+1` (`CFBundleShortVersionString=0.0.0`, `CFBundleVersion=1`).
10. Настрой понятное имя iOS-приложения, портретную ориентацию, тесты, `flutter analyze` и `flutter test`.
11. Используй предоставленный пользователем логотип как iOS AppIcon и в экране «О приложении».
12. Не изменяй исходные Python-проекты и не перезаписывай пользовательские изменения.

Выполни промт полностью, а не ограничивайся планом.
