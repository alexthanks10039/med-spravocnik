# Справочник врача: UI/UX и Flutter-архитектура

## 1. Структура экранов

Основная навигация адаптивная: `NavigationBar` на смартфоне и `NavigationRail` на вебе/планшете.

- Главная: глобальный поиск, быстрые категории, недавно открытые материалы, статус offline-библиотеки.
- Глобальный поиск: debounce, подсказки, смешанная выдача заболеваний, препаратов и калькуляторов.
- Категории: заболевания, препараты, калькуляторы, клинические рекомендации и неотложная помощь.
- Заболевания: список по специальности и детальная страница с блоками «Кратко», «Диагностика», «Красные флаги», «Лечение».
- Препараты: список и монография с показаниями, дозированием, противопоказаниями, коррекцией дозы и взаимодействиями.
- Калькуляторы: каталог и рабочие формы BMI и CKD-EPI 2021.
- Сохранённое: избранное, история и заметки.
- Профиль: специальность, профессиональные интересы и сведения о приложении.
- Настройки: тема, offline-библиотека, обновления, предупреждения и приватность.
- Состояния: `loading`, `offline`, `error`, `empty`, `not found`.

Ключевой путь: запуск → поиск → результат → структурированный материал → избранное / копирование / заметка / связанный материал.

## 2. Дизайн-система

- Основа: Material 3 с собственными токенами и спокойной клинической плотностью.
- Цвета: deep navy `#092B3C`, teal `#087F7A`, cyan `#37B9C6`, graphite `#172127`, light canvas `#F4F7F7`.
- Семантика: teal — основное действие; cyan — информационные акценты; amber — внимание; red — красные флаги и критические предупреждения.
- Поверхности: белые/графитовые карточки, радиус 20 px, тонкая рамка вместо тяжёлых теней.
- Типографика: крупный заголовок 32–40 px, заголовки секций 20–24 px, основной текст 15–16 px с высотой строки 1.5.
- Сетка: базовый шаг 4 px; основные интервалы 8, 12, 16, 20, 24 и 32 px.
- Доступность: контраст не ниже WCAG AA, кликабельная область от 44 px, смысл не передаётся только цветом.
- Темы: светлая, тёмная и системная; выбор сохраняется локально.

## 3. Компоненты

- `AppShell`: адаптивная навигационная оболочка.
- `ScreenFrame`: единая ширина контента и large app bar.
- `ClinicalSearchField`: первичный поисковый паттерн.
- `MedicalItemCard`: унифицированный результат поиска и элемент списка.
- `SectionHeader`: заголовок раздела с контекстным действием.
- `StatePanel`: empty/error/offline/not-found состояния.
- `ClinicalAlert`: красные флаги и предупреждения.
- `CalculatorCard`: форма, валидация, результат и клинический disclaimer.
- `RelatedItem`: переход к связанному заболеванию, препарату или расчёту.
- `CategoryCard`, `StatusChip`, `DoseBlock`, `EvidenceBadge` — следующие компоненты для production-версии.

    search/presentation/
    catalog/presentation/

Для production каждая feature расширяется папками `domain`, `data`, `application`, `presentation`. Репозиторий скрывает remote API и локальную БД от UI.

## 6. MVP-приоритеты

1. Поиск, заболевания, препараты, детальные страницы и offline-кэш.
2. Избранное, история, заметки и два наиболее востребованных калькулятора.
3. Авторизация, синхронизация между устройствами и обновление медицинского контента.
4. Расширенный drug-interaction checker, клинические алгоритмы и персонализация.
5. Evidence grading, редакционный workflow и аудит изменений контента.

## 7. Рекомендации по реализации

- Local database должна быть источником чтения для UI; сеть обновляет локальные данные в фоне. Для production подойдут Drift/SQLite или Isar.
- API DTO не должны попадать в presentation. Используйте mapper между DTO, local entity и domain model.
- Синхронизацию вести по `updatedAt`, версии контента и tombstone-записям для удалений.
- Поиск: локальный FTS для offline и серверный ранжированный поиск при наличии сети; результаты объединять без скачков интерфейса.
- Медицинские статьи хранить структурированными блоками, а не одним Markdown-полотном. Это упрощает копирование, заметки и обновление отдельных секций.
- Калькуляторы описывать декларативной схемой, а формулы покрывать unit-тестами на опубликованных контрольных примерах.
- Добавить `freezed`/`json_serializable` после стабилизации API-контрактов, а не раньше.
- Для реального релиза обязательны контентная версия, дата пересмотра, источник, уровень доказательности и медицинский disclaimer.
- Настроить golden-тесты для светлой/тёмной темы и размеров 390, 768 и 1440 px.
