# Разработка

## Правила изменений

- Не смешивайте OCR ingestion и исполняемые калькуляторы.
- Не меняйте raw OCR-поля ради отображения; используйте presentation layer.
- Не добавляйте медицинский корпус, логи и локальные индексы в Git.
- Новые API должны иметь Pydantic-модель и тест контракта.
- Preview использует только synthetic/public demo data.
- Содержимое `archive` не исправляется вместе с активным приложением.

## Проверки перед коммитом

```powershell
.\scripts\test.ps1
git status --ignored
git diff --check
```

Публичный прогон Calculator API исключает только
`test_hf_benchmark_dataset.py`, поскольку этот upstream-тест требует закрытый
hidden-test bundle. Все остальные калькуляторные тесты остаются обязательными.

Дополнительно проверьте отсутствие секретов:

```powershell
git grep -n -I -E "(BEGIN .*PRIVATE KEY|ghp_|github_pat_|sk-[A-Za-z0-9])"
```

## Работа с корпусом

SQLite должна оставаться read-only для runtime API. Изменение схемы и данных
выполняется pipeline с отдельным отчетом качества и версией корпуса.

## Ветки

- `main`: проверенная активная версия;
- `codex/*`: изолированные изменения и миграции;
- Preview хранится в каталоге, а не заменяет production-конфигурацию.
